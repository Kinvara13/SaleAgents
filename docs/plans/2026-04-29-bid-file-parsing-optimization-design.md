# 招标文件解析优化设计文档

**日期**：2026-04-29  
**状态**：已批准  
**目标**：80%+ 准确度（初稿，需人工审核）

---

## 1. 需求背景

### 1.1 当前问题
1. **字段识别不准确** - 该提取的没提取到
2. **字段匹配错位** - A字段内容填到B字段
3. **星标项无法识别** - * 或 ★ 符号
4. **评分标准提取不准确** - 关键业务逻辑
5. **提取内容不完整** - 只提取部分
6. **解析速度太慢** - 用户体验差

### 1.2 目标字段
- 项目名称
- 招标编号
- 投标截止时间
- 评标标准/评分细则
- 技术要求/规格参数
- 资质要求
- 投标人须知
- 合同条款
- 商务条款（付款方式、保证金等）
- 废标条款
- 星标项（* 或 ★）

### 1.3 文件类型
- PDF（文字版）
- Word (.docx)
- Excel (.xlsx)

---

## 2. 解决方案

### 2.1 方案选择
**方案A：规则预提取 + LLM 补充提取（混合方式）**

- 规则层：快速、确定性提取（项目名称、招标编号、截止时间等结构化字段）
- LLM层：语义理解、复杂字段提取（评分标准、技术要求等非结构化字段）
- 星标项：规则识别 + LLM 补充

### 2.2 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│  ZIP 解压 → 文件识别                                          │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  文本提取层（PDF/DOCX/XLSX → 纯文本）                         │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  规则预提取层（新增）                                          │
│  - 项目名称、招标编号、截止时间（正则）                          │
│  - 星标项识别（正则 + 位置感知）                                │
│  - 评分标准、废标条款（关键词匹配）                             │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  LLM 补充提取层（优化提示词）                                  │
│  - 技术要求、资质要求、合同条款等                               │
│  - 星标项补充识别                                             │
│  - 字段验证和补充                                             │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  字段匹配层（改进路由）                                        │
│  - 规则提取结果优先                                           │
│  - LLM 提取结果补充                                           │
│  - 合并去重、冲突解决                                          │
└─────────────────────────────────────────────────────────────┘
```

## 3. 实施计划

### 3.1 新增模块：`rule_based_extractor.py`

**文件位置**：`backend-v2/app/services/rule_based_extractor.py`

**功能**：
1. 项目名称提取（正则 + 关键词）
2. 招标编号提取（正则）
3. 投标截止时间提取（正则）
4. 星标项提取（正则 + 位置感知）
5. 评分标准提取（章节匹配）
6. 废标条款提取（关键词匹配）

**输入**：纯文本 + 文件类型  
**输出**：结构化字段字典

### 3.2 优化模块：`llm_parsing_client.py`

**改动**：
1. 优化字段提取提示词
2. 增加星标项提取指令
3. 改进输出格式（JSON）

### 3.3 优化模块：`asset_routing_service.py`

**改动**：
1. 增加字段-章节映射
2. 优先级匹配（规则 > LLM）
3. 合并去重逻辑

### 3.4 星标项处理

**识别规则**：
- 正则：`[★\*]\s*(.+?)(?=\n|$)`
- 位置感知：只在特定章节中提取
- 存储：单独字段 `star_marked_items`

**出现场景**：
- 评分标准
- 资质要求
- 废标条款
- 技术要求

## 4. 测试验证

### 4.1 测试用例
1. 结构完整的招标文件（章节清晰）
2. 结构混乱的招标文件（章节不清晰）
3. 包含星标项的招标文件
4. 超长招标文件（>20K 字符）
5. 包含表格的招标文件

### 4.2 验收标准
- 字段识别准确率 ≥ 80%
- 星标项识别准确率 ≥ 90%
- 解析速度 < 30秒/文件
- 无字段匹配错位

## 5. 风险与依赖

### 5.1 风险
1. 正则规则覆盖不全 → 持续优化规则库
2. LLM 提取不稳定 → 增加规则兜底
3. 星标项格式多样 → 扩展识别规则

### 5.2 依赖
- 无新增外部依赖
- 使用现有 pypdf、pdfplumber、python-docx、openpyxl
