# 招投标智能体页面原型与字段清单

## 1. 文档信息

- 文档名称：招投标智能体页面原型与字段清单
- 版本：V1.0
- 日期：2026-03-18
- 关联文档：`docs/bid-agent-prd.md`、`docs/bid-agent-architecture.md`
- 目标读者：产品经理、前端工程师、后端工程师、测试工程师、业务验收人

## 2. 文档目的

本文件用于补齐 PRD 中的页面层设计，明确：

- 系统有哪些页面
- 每个页面解决什么问题
- 页面核心区域如何布局
- 页面需要哪些字段
- 关键交互和状态是什么
- 哪些位置体现 AI 能力和人工复核

本文件按 MVP 优先，优先覆盖投标决策、标书生成、材料审核三大模块及其共用后台能力。

## 3. 页面地图

```text
登录
-> 首页工作台
-> 项目管理
   -> 项目列表
   -> 新建项目
   -> 项目详情
   -> 文档解析校正
   -> 投标决策
   -> 标书生成
   -> 材料审核
   -> 审批记录
   -> 导出中心
-> 知识库管理
   -> 资产列表
   -> 资产详情
   -> 检索测试
-> 规则中心
   -> 决策规则
   -> 审核规则
   -> 红线条款
   -> AI 任务配置
-> 系统设置
   -> 角色权限
   -> 模型与接口配置
```

## 4. 全局设计原则

- 左侧导航承载一级模块，顶部承载搜索、通知、用户信息
- 项目相关页面统一使用项目头部信息条
- 所有 AI 输出区域均展示引用依据、置信度或待确认标记
- 风险与问题统一使用分级颜色和状态标签
- 支持“系统建议”和“人工最终确认”并存

## 23. 字段设计补充原则

- 所有金额字段统一币种
- 所有日期字段统一到时区和格式
- 风险等级统一为 P0/P1/P2/P3
- AI 输出字段需带置信度和引用来源
- 审批相关字段需保留操作时间和操作者

## 25. 前后端联调重点

- 文档预览与字段定位的坐标体系需统一
- 决策评分卡要支持规则结果和 AI 解释同时展示
- 章节编辑器要支持材料引用插入与待确认标记
- 审核问题要支持从问题列表反查到原文位置
- AI 输出组件需统一展示引用、置信度和人工复核入口
