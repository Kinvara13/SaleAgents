# SaleAgents 测试规范

## 1. 测试层级

### 静态审计

- 路由是否存在
- 页面是否接线到 service
- service 是否接线到真实 API
- 文档是否与代码一致

### API 烟测

- 使用 Python 3.11 启动 `backend-v2`
- 用真实 token 调关键接口
- 结果写入 `memory-bank/TEST_REPORT_V2.md`

### 页面走查

- 先手工，再决定是否补浏览器自动化
- 每个页面至少验证进入、加载、核心动作、错误态

## 2. 环境要求

- Python：`3.11`
- Node：可运行 `npm run build`
- 默认后端地址：`http://127.0.0.1:8000`
- 默认账号：`admin / admin123`

## 3. 测试记录要求

所有测试结果都必须写明：

- 测试日期
- 环境
- 被测接口或页面
- 输入
- 输出
- 结论
- 是否阻塞

## 4. 缺陷分级

- `P0`
  - 主流程不可用
  - 关键接口 500
  - 页面无法完成核心动作
- `P1`
  - 功能可用但闭环缺失
  - 规则、素材、工作台等关键能力缺失
- `P2`
  - 展示、文案、次级交互问题

## 5. 本项目当前必测清单

- 登录鉴权
- 项目 CRUD
- 招标信息列表

- 解析章节列表

## 6. 自动化引入门槛

只有满足以下条件后，才考虑扩大浏览器自动化范围：

- 页面契约稳定
- API 契约稳定
- 至少完成一轮人工走查
- 不再频繁出现 axios/fetch 这类基础接线错误
